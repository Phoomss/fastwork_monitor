# Fastwork Jobboard Monitor for Discord 🚀 (Node.js version)

สคริปต์นี้จะทำหน้าที่ตรวจสอบงานใหม่บนบอร์ดงานของ Fastwork (Jobboard) แบบอัตโนมัติ และส่งการแจ้งเตือนรูปแบบการ์ดที่สวยงาม (Discord Embed Card) ไปยัง Discord Channel ของคุณโดยทันทีผ่าน **Discord Webhook**

---

## ฟีเจอร์เด่น (Features)
- **ระบบคัดกรองงานเฉพาะทาง (Targeted Filters):** กรองและแจ้งเตือนเฉพาะงานด้าน **การพัฒนาเว็บไซต์ (Web Dev)**, **การพัฒนาแอปพลิเคชัน (App Dev)** และ **การออกแบบ UX/UI** เท่านั้น โดยคัดกรองจากหมวดหมู่และคีย์เวิร์ดที่แม่นยำเพื่อป้องกันสแปมหรือ false positives
- **ไม่ต้องลง library เพิ่มเติม:** ใช้โมดูลมาตรฐานของ Node.js (HTTPS) ทำงานได้ทันทีโดยไม่ต้องสั่ง `npm install`
- **ระบบทำงานในเครื่องคอมพิวเตอร์ทั่วไป (Console-Only Fallback):** หากไม่ได้กรอก Webhook ระบบจะสลับไปรันแบบแสดงผลที่หน้าจอเทอร์มินัลของคุณให้อัตโนมัติ เพื่อให้สามารถลองใช้ได้ทันที
- **ระบบป้องกันการแจ้งเตือนซ้ำ:** เก็บประวัติงานที่แจ้งเตือนไปแล้วลงในไฟล์ `seen_jobs.json`
- **ระบบคัดกรองข้อมูลประวัติการทำงานครั้งแรก:** ในการเปิดรันครั้งแรก ระบบจะไม่สแปมงานเก่าทั้งหมดลง Discord แต่จะเริ่มแจ้งเตือนเฉพาะงานใหม่ที่โพสต์หลังจากนั้น
- **ทดสอบระบบง่าย:** มีคำสั่งสำหรับส่งการแจ้งเตือนทดสอบ (Test Mode) เข้า Discord ทันที
- **สองรูปแบบการทำงาน:**
  - `Loop Mode` (รันบอทค้างไว้คอยตรวจสอบตลอดเวลา)
  - `Once Mode` (รันเพียงครั้งเดียวแล้วจบ เหมาะสำหรับติดตั้งร่วมกับ **Cron Job** บน Mac)

   - **ภาษาอังกฤษ (เช็คตรงคำ):** ios, android, flutter, react native, kotlin, swift, mobile app

## ขั้นตอนการติดตั้งและการใช้งาน (Setup Guide)

### 1. คัดลอกและตั้งค่าไฟล์ `.env`
ให้คัดลอกไฟล์ `.env.example` เป็น `.env` โดยเปิดเทอร์มินัลในโฟลเดอร์นี้แล้วรันคำสั่ง:
```bash
cp .env.example .env
```

เปิดไฟล์ `.env` แล้วแก้ไขค่า:
- `DISCORD_WEBHOOK_URL` -> ใส่ Webhook URL ของช่อง Discord ของคุณ
  *(วิธีสร้าง Webhook: ไปที่ช่องแชท Discord -> คลิกไอคอนฟันเฟือง (Settings) -> Integrations -> Webhooks -> Create Webhook -> Copy Webhook URL)*
- `CHECK_INTERVAL_SECONDS` -> ตั้งค่าความถี่ในการตรวจสอบงาน (แนะนำ: 60 วินาที)

---

### 2. วิธีรันเพื่อทดสอบบอท (Test Mode)
คุณสามารถสั่งให้บอทดึงงานล่าสุด 1 งานและแสดงผลทันที เพื่อทดสอบการทำงานของฟิลเตอร์และตรวจสอบ Discord:
```bash
node fastwork_monitor.js --test
```
*(หากตั้งค่า Webhook ในไฟล์ `.env` แล้ว ข้อมูลจะยิงเข้าดิสคอร์ดทันที แต่หากยังไม่ได้ตั้งค่า ระบบจะแสดงผลข้อความการ์ดในหน้าจอเทอร์มินัลแทน)*

---

### 3. วิธีการรันใช้งานจริง (Execution Modes)

#### แบบที่ A: รันแบบ Loop ค้างไว้ตลอดเวลา (Loop Mode)
เหมาะสำหรับการเปิดเทอร์มินัลทิ้งไว้ หรือรันบน Server/VPS:
```bash
node fastwork_monitor.js
```
*(บอทจะตรวจสอบงานใหม่ทุกๆ 60 วินาทีตามที่ตั้งค่าไว้ในไฟล์ `.env`)*

#### แบบที่ B: ตั้งเวลาทำงานผ่าน Cron Job (สำหรับผู้ใช้ Mac)
หากคุณใช้ Mac และต้องการให้บอททำงานเงียบๆ ในเบื้องหลังโดยไม่ต้องเปิดหน้าต่างเทอร์มินัลค้างไว้ คุณสามารถใช้ **Cron** ของเครื่องในการสั่งรันทุกๆ 5 นาทีได้:

1. เปิดหน้าต่างตั้งค่า Cron:
   ```bash
   crontab -e
   ```
2. ใส่บรรทัดนี้ลงไป (ตรวจสอบและแก้ไขพาธของโฟลเดอร์ให้ตรงกับเครื่องของคุณ โดยต้องระบุพาธเต็มของ Node.js เช่น `/opt/homebrew/bin/node`):
   ```cron
   */5 * * * * cd /Users/mac/Desktop/workspace/fastwork_varidation && /opt/homebrew/bin/node fastwork_monitor.js --once >> monitor.log 2>&1
   ```
3. บันทึกและปิด (ระบบจะทำการเช็คงานใหม่ให้คุณโดยอัตโนมัติทุกๆ 5 นาที)
   *(หมายเหตุ: บน macOS คุณต้องไปที่ System Settings -> Privacy & Security -> Full Disk Access แล้วเพิ่มและเปิดสิทธิ์ให้แก่ `/usr/sbin/cron` เพื่ออนุญาตให้ Cron เข้าถึงไฟล์ในโฟลเดอร์ต่างๆ ได้)*

#### แบบที่ C: รันผ่าน Docker (แนะนำสำหรับสาย Container)
หากเครื่องหรือเซิร์ฟเวอร์ของคุณติดตั้ง Docker และ Docker Compose ไว้แล้ว สามารถสั่งรันได้ง่ายๆ:

1. เปิดแอปพลิเคชัน Docker Desktop ในเครื่องให้เรียบร้อย
2. บิวด์และเริ่มรันบอทในเบื้องหลัง (Detached Mode):
   ```bash
   docker compose up -d --build
   ```
3. ตรวจสอบสถานะและดู Log การทำงาน:
   ```bash
   docker compose logs -f
   ```
4. หยุดการทำงานของบอท:
   ```bash
   docker compose down
   ```
*(ระบบ Docker จะเชื่อมโยงไฟล์ `seen_jobs.json` ระหว่างคอมพิวเตอร์ของคุณกับตัวคอนเทนเนอร์โดยอัตโนมัติ เพื่อรักษารายการประวัติงาน และโหลดค่าคอนฟิกจากไฟล์ `.env` เข้าสู่คอนเทนเนอร์ทันที)*

#### แบบที่ D: รันในเบื้องหลังด้วย PM2 (แนะนำที่สุดสำหรับ macOS / Server 🌟)
หากต้องการรันบอททิ้งไว้แบบถาวรในเบื้องหลัง แม้จะปิดหน้าต่าง Terminal ไปแล้ว บอทก็ยังทำงานอยู่ และช่วยกู้ชีพบอทให้กลับมาทำงานใหม่ทันทีหากบอทดับหรือมีข้อผิดพลาด:

1. สั่งรันบอทผ่าน PM2:
   ```bash
   npx pm2 start fastwork_monitor.js --name "fastwork-monitor"
   ```
2. ตรวจสอบสถานะการทำงานทั้งหมด:
   ```bash
   npx pm2 status
   ```
3. ดูข้อมูล Log และประวัติการทำงานแบบ Real-time:
   ```bash
   npx pm2 logs fastwork-monitor
   ```
4. หยุดการทำงานชั่วคราว:
   ```bash
   npx pm2 stop fastwork-monitor
   ```
5. ลบออกจากระบบจัดการของ PM2:
   ```bash
   npx pm2 delete fastwork-monitor
   ```

## โครงสร้างไฟล์
- `fastwork_monitor.js` - สคริปต์หลักของบอท (Node.js)
- `.env` - ไฟล์เก็บค่าคอนฟิกูเรชัน (Discord Webhook และพารามิเตอร์อื่นๆ)
- `seen_jobs.json` - ไฟล์เก็บประวัติรหัสงานที่ตรวจพบแล้ว (สร้างขึ้นอัตโนมัติเมื่อบอทรันครั้งแรก)
